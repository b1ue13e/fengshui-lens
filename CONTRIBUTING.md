# CONTRIBUTING

感谢你愿意给 **青年大学习** 提交贡献。

这个项目的目标不是做一个“功能越来越多的内容站”，而是做一个：

- 真正帮用户少踩坑的现实决策产品
- 有产品感、信息架构感和交互节奏的中文开源项目
- 值得被更多开发者、设计师、独立开发者和产品人关注的 **10k stars 级别仓库**

所以这里欢迎的不只是代码贡献，也欢迎：

- 产品设计建议
- 信息架构优化
- 文案与 onboarding 改写
- UI / UX 改进
- README / 演示 / 传播素材
- 场景训练内容补全
- 测试与稳定性修复

---

## 一句话原则

> 每一次贡献，都应该让用户更快进入主线、更少靠猜、更容易继续行动。

---

## 我们最欢迎的贡献类型

### 1. 产品体验优化
例如：

- 首页首屏转化更强
- 租房判断链路更顺
- compare / report / 到沪承接更清楚
- 搜索、分类、场景详情更不绕

### 2. 前端 UI / UX 优化
例如：

- 提升视觉一致性
- 提高移动端体验
- 修复信息层级混乱
- 优化卡片、按钮、列表、表单体验

### 3. 内容与场景训练补全
例如：

- 新的训练场景
- 更好的 FAQ
- 更能直接带走的话术模板
- 更有用的 checklist

### 4. 工程质量提升
例如：

- 修 bug
- 提高稳定性
- 增强类型约束
- 测试覆盖
- 仓储 / 引擎 / 路由结构整理

---

## 不太建议的贡献方向

以下方向请先开 issue 讨论，不建议直接 PR：

- 大规模重写所有 UI
- 脱离主线的功能堆叠
- 把首页改成资源门户
- 做成泛内容站 / 泛资讯站
- 一次性引入大量基础设施复杂度
- 没有验证需求的“平台化扩张”

## 本地开发

### 1. 安装依赖

```bash
npm install
```

### 2. 启动开发环境

```bash
npm run dev
```

### 3. 提交前至少执行

```bash
npm run lint
npm run build
```

如果你改了核心判断逻辑，建议额外跑：

```bash
npm run test:run
```

如果你改了路由或页面承接，建议额外跑：

```bash
npm run smoke:routes
```

## 分支与提交建议

### 分支命名
推荐：

- `codex/feature-xxx`
- `codex/fix-xxx`
- `codex/readme-xxx`
- `codex/ui-xxx`

### commit message
推荐使用清晰的英文前缀：

- `feat:` 新功能
- `fix:` 修复
- `docs:` 文档
- `refactor:` 重构
- `test:` 测试
- `chore:` 杂项

例如：

- `feat: improve rent compare handoff`
- `fix: stabilize scenario detail state`
- `docs: upgrade readme hero and screenshots`

---

## PR 要求

请尽量让一个 PR 做一件事。

### 好的 PR 应该具备：

- 改动范围清楚
- 改动理由明确
- 没有顺手改很多无关文件
- 给出验证方式
- UI 改动最好附截图

### PR 描述建议包含：

```md
## What
这次改了什么

## Why
为什么要改

## Verification
如何验证

## Notes
还有哪些限制 / 后续可以继续做什么
```

---

## UI / 产品设计规则

这个项目非常看重 **产品收口能力**。提交 UI 或内容改动前，请优先判断它是不是在帮助主线更清楚，而不是更热闹。

### 首页规则
首页必须：

- 保持租房判断为第一主任务
- 让“青年大学习”品牌尽早出现
- 不让资源导航占据第一屏主导
- 让 copy 明确连接：
  - master brand
  - rental-first entry
  - Shanghai settlement handoff

### 产品表达规则
优先做：

- 先判断，再补证据，再继续动作
- 先顺序，再入口
- 先高频问题，再扩展内容

避免：

- 门户式堆入口
- 信息太满但主任务不清楚
- 只有 UI 好看，没有产品节奏

## 内容贡献格式建议

如果你想新增一个训练场景，建议至少准备：

- 标题
- 一句话摘要
- 适合谁
- 去之前先准备
- 按步骤怎么做
- 常见坑
- FAQ
- 可复制话术
- checklist
- 小测验

目标不是“写一篇内容”，而是“做一张能继续带用户行动的训练页”。

---

## 如果你想做大改

如果你想改：

- 首页结构
- 租房主链
- compare / report 核心结构
- 内容模型
- 场景数据结构

请优先：

1. 开 issue
2. 写清楚问题是什么
3. 说明你想怎么改
4. 说明为什么这对产品主线更好

---

## 优先级最高的问题

如果你不知道该贡献什么，优先看这几类：

- 能提升首页主线转化的改动
- 能减少用户理解成本的改动
- 能增强 compare / report / handoff 承接的改动
- 能让 README / 截图 / 架构表达更强的改动
- 能明显修复体验 bug 的改动

---

## 最后

如果这个项目真的要走到 **10k stars**，靠的不是“再加几十个功能”，而是：

- 更像产品
- 更像作品
- 更有记忆点
- 更有传播性
- 更容易让人一眼看懂“它为什么值得 star”

欢迎一起把它做成一个真正有产品感的中文开源项目。
